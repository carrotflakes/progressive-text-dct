# 実験タスク3: 文脈化エンコーダ + DCT vs 学習レイテント(プログレッシブ・テキスト圧縮)

## 背景とゴール

task2(フルスクラッチ版)の結論:
- H1/H2は実証された(K=64でほぼ完全復元、小Kで大意保存)
- しかし**スペクトル適応は起きなかった**: 文脈化なしの静的埋め込みでは、系列方向の周波数分布はトークン列の自己相関で決まってしまい、埋め込みテーブルにはスペクトルを傾ける自由度がない
- B3(ランダムK係数)が小Kの自由生成でmainに勝った = 固定の低周波選択は小K予算で最適でない

そこで task3 では2つの問いに決着をつける:

1. **Q-A (スペクトル適応)**: 埋め込みの後に双方向Transformerエンコーダを置き、DCT切り詰めをend-to-endで通すと、エンコーダは「低周波に情報を寄せた滑らかな表現」を学習するか?(spectrum.pngで直接観測)
2. **Q-B (DCT必要性)**: 粗→細の順序付けをDCT(固定基底)に外注する必要はあるか? cross-attentionで集約した**学習レイテント列+prefix-drop学習**(順序は学習で獲得)はDCTを上回るか?

## 手法

task2のコードベース(progressive-text-dct-scratch/)を拡張する。共通部分(トークナイザ、データ、デコーダ、評価)は同一。

### 共通設定

- トークナイザ/データ: task2と同一(BPE16k、wikitext-103全量、32-128トークンチャンク)
- デコーダ: task2と同一(Pre-LN Transformer d512・8層・8ヘッド・FFN2048・RoPE、prefix-LM)
- K_max=64、Kはlog-uniformサンプリング(prefix-drop学習)
- シード42、config.yamlに集約

### バリアント(3本、同一学習予算・同一オプティマイザ)

- **A (enc-dct)**: E → 双方向Transformerエンコーダ(d512・4層・8ヘッド・FFN2048、RoPE、パディングマスク) → 系列方向DCT → 先頭K係数。勾配はエンコーダとEまで貫通
- **B (enc-latent)**: 同じエンコーダ出力に対し、M=64個の学習可能クエリがcross-attentionで集約し、レイテント列 L ∈ R^{64×512} を生成。先頭Kレイテントのみ保持(prefix-drop)。順序付けは学習圧力のみで獲得させる。DCTは使わない
- **C (emb-dct)**: task2のmain(Eのみ→DCT)を同一予算・同一オプティマイザで再学習。エンコーダ効果とオプティマイザ効果を分離する対照

### オプティマイザ: Muon(ハイブリッド)

- 隠れ層の2D重み行列(attention/FFN/projection)→ **Muon**(momentum 0.95、Newton-Schulz 5回、Moonlight方式のRMSマッチング `0.2·√max(m,n)` スケール)
- 埋め込み類(E、デコーダtok emb、idx/len emb)、出力ヘッド、LayerNorm、bias、BOS、レイテントクエリ → **AdamW**(従来どおり)
- lr/スケジュールは両グループ共通: 3e-4、warmup 2000、cosine decay、wd 0.1
- **導入前にパイロット必須**: task2 main構成で AdamW vs Muon を3000step比較し、損失改善を実測してから採用を確定する。効果がなければAdamWのまま進める(その旨を記録)

### 学習

- **60,000 step・実効バッチ64**(Muonのサンプル効率向上を見込みtask2の100kから短縮。パイロット結果次第で調整し、3本同一予算を維持)
- チェックポイント15分ごと・再開可能、tmuxセッションで実行

### サニティチェック

- 既存ユニットテスト(DCT可逆性等)を維持
- 新規: A/Bともに勾配がエンコーダ・Eまで到達することのテスト、Bのレイテントprefix-drop整合性テスト
- 100サンプル過学習(K=64固定)で復元精度~100%(AとBそれぞれ)

## 評価

test 2000件、K ∈ {1,2,4,8,16,32,64}、task2と同一指標(token acc / 完全一致 / BLEU / chrF / 意味類似度)+同一の定性表。

- 曲線図: A, B, C + task2のmain(参考、別オプティマイザの旨明記)を重ねる
- **スペクトル分析(本実験の主役)**: Aのエンコーダ出力のDCTエネルギー分布をtask2と同形式で可視化。低周波集中が起きていればQ-Aの直接証拠
- Bの順序性検証: レイテントごとの寄与(K=1..64の増分品質)が「先頭ほど粗い情報」になっているか
- 仮説判定:
  - **H-A**: Aはスペクトル低周波集中を獲得し、小K品質(semsim)でCを上回る
  - **H-B**: BはAを上回る(固定基底DCTより学習順序付けが有利)。逆ならDCTの外注に価値がある証拠
  - **H-C**: Muonは同一構成(C vs task2 main)で学習を高速化する(60kでtask2の100k相当以上)

## 進め方と制約

- Phase 0: Muon実装+パイロット(AdamW vs Muon 3000step実測) → Phase 1: A/B実装+サニティ → Phase 2: 本学習3本(tmux) → Phase 3: 評価・スペクトル・レポート
- 学習1本あたり数時間以内。超えそうならステップ数を3本一律に縮小し明記
- 評価指標は既存ライブラリを使用(task2と同一コード)
