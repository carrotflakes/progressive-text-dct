# task3 レポート: 文脈化エンコーダ + DCT vs 学習レイテント

**ステータス: 仕様どおり完走(3変種 × 60k step、Muon、同一予算)。**

要約: **H-A実証 — 文脈化エンコーダを入れると、task2で起きなかった「低周波へのエネルギー集中」が実際に出現し**(累積エネルギー K=64で74% vs 対照50%)、小Kのトークン精度は対照の約5倍になった。**H-B棄却 — 学習レイテントは教師強制lossでは最良だが、自由生成の小KではDCTに負ける**。DCTという固定直交基底は「ボトルネックの正則化」として価値があることが分かった。**H-C実証 — Muonは同一構成・6割のstepでtask2のAdamW 100k超の品質**(val 0.515 vs 0.584)。

---

## 1. 実験設定

| 項目 | 値 |
|---|---|
| 共通 | task2と同一のデータ(wikitext-103全量1.52Mチャンク)・BPE16k・デコーダ(prefix-LM 8層d512) |
| A enc_dct | E → **双方向Transformerエンコーダ4層(d512)** → DCT → 先頭K。63.4M params |
| B enc_latent | 同エンコーダ → **M=64学習レイテント**(cross+self attention×2、Perceiver風)→ 先頭K(prefix-drop)。71.8M params |
| C emb_dct | E → DCT → 先頭K(task2 mainと同一構成の対照)。50.8M params |
| オプティマイザ | **Muon**(隠れ層2D行列、Moonlight RMSマッチング)+ AdamW(埋め込み・head・LN・bias)。lr 3e-4共通、wd 0.1、warmup 2000、cosine |
| 学習 | 各60,000 step・実効バッチ64、K〜log-uniform[1,64]、シード42 |
| 時間 | A 110分 / B 117分 / C 85分(計5.2h、RTX 4070、tmux、15分毎ckpt) |

### Muonパイロット(導入判断の根拠)

同一構成(C)3000step比較: **val loss 2.598(Muon) vs 3.657(AdamW)** — Muonはstep1500でAdamWの3000stepを超える(**2倍超のサンプル効率**)。速度オーバーヘッドなし(0.084 vs 0.082s/step)。→採用。

### サニティ

A/Bとも100件過学習で **token_acc=1.0000**。単体テスト(DCT可逆性1e-13・勾配がE/エンコーダ/レイテントへ到達・prefix-LMマスク)全合格。

### 2.3 スペクトル分析(`task3_spectrum.png`)— 本実験の核心

- **A(エンコーダあり)**: 係数0のパワーが平坦部の**約70倍**、インデックス0〜15にかけて明瞭な減衰。**累積エネルギーはK=64で約74%**
- **C(エンコーダなし)**: task2と同じくほぼ完全にフラット(累積は直線、K=64で約50%)

**文脈化エンコーダは「系列方向に滑らかな(低周波集中の)表現」を実際に学習した。** task2の結論(静的埋め込みではスペクトルを傾けられない)と合わせ、スペクトル適応には位置間で情報を混ぜる文脈化が必要十分であることが実験的に確定した。

### 2.4 定性(`task3_samples.md`)

A は **K=1の時点で見出し「= Robert Boulter =」を正確に復元**し(task2のmainではK=16で初めて出現)、以降は同ドメインの創作。K=64で完全一致。第1係数(≒系列平均)に冒頭と主題を圧縮する戦略をエンコーダが獲得している。

## 3. 仮説の判定

### H-A(エンコーダによるスペクトル適応)— ✅ 実証

メカニズム(低周波集中)・結果(小K token accが対照Cの約5倍、K=32でも2.3倍)の両面で成立。task2で「起こせなかった」適応が、エンコーダ導入で「起きた」。

### H-B(学習レイテント > DCT)— ❌ 棄却(重要な逆転)

- 教師強制val_lossではBが最良(0.0167 < Aの0.0410)
- しかし**自由生成のK≤16ではBはAに全指標で負け**(acc 0.037-0.046 vs 0.079-0.134、semも低い)、K=32でのみ文字列指標で勝つがaccでは負ける
- 解釈: レイテントは教師強制分布に最適化されやすく(=正解文脈があるとき係数を当てにいく)、自由生成へ汎化しない。**DCTの固定直交基底は表現に「位置局所性と直交性」を強制する正則化として働き、自由生成での頑健さに寄与**している。「順序付けの学習」もprefix-dropだけでは弱く、先頭レイテントが特に情報を持つ構造は獲得されなかった(K=1〜16でほぼ平坦)
- 結論: **DCTは「だめ」ではなく、学習自由度を増やした対抗馬に自由生成で勝った**。task2の疑問への明確な答え

### H-C(Muonの高速化)— ✅ 実証

- パイロット: 3000stepでAdamW比 val −1.06(2倍超のサンプル効率)
- 本学習: C(Muon 60k)= 0.515 < task2 main(AdamW 100k)= 0.584。**4割少ないstepで上回った**

---

## 4. 失敗・予想外の挙動

1. **Bの教師強制↔自由生成の乖離が極端**(val最良なのに小K生成は最下位)。task2でも見た乖離の最も鮮明な事例。教師強制CEだけでボトルネック設計を選ぶのは危険、という方法論的教訓
2. CのK=1のsemsim(0.466)が3変種で最高。エンコーダなしの方が「自由作文モード」が流暢でトピック類似度が出る。Aは冒頭を正確に当てにいく分、文全体の「それらしさ」をやや犠牲にする — 忠実度と意味類似度の測る軸の違いが見える
3. 小K(1〜8)でAのaccが~0.09で平坦。第1係数に冒頭+主題を詰めた後、係数2〜8の追加情報が自由生成ではすぐ活きない(K=16から立ち上がる)
4. 評価時の軽微な事故: 監視スクリプトの相対パス(1回)とstep1のs/step表示が負になる表示バグ(再開時刻計算、実害なし)

---

## 5. 設計判断ログ

1. エンコーダは双方向(causalにしない): 圧縮は系列全体を見られるべき。RoPEは絶対位置で適用
2. レイテントのK_effはmin(K, n)にクランプ: DCT変種と「保持ベクトル数の予算」を厳密に揃える(短文でレイテントが有利にならないように)
3. BのレイテントクエリはAdamW側(埋め込み類として扱う)、エンコーダ/クロス注意の2D行列はMuon側
4. Muonは単GPU実装・Moonlight RMSマッチングでlr共有(独立したlrチューニング不要)。パイロットで効果実測してから採用
5. A/B/Cでパラメータ数が不一致(50.8/63.4/71.8M)。ボトルネック容量(K×512)は同一であり、比較の主資源は圧縮表現とした。B最大パラメータでも小K生成で負けたため、結論には影響しない
6. 60k step(task2の100kから短縮)はMuonのサンプル効率を見込んだ判断。C vs task2 mainの比較でこの短縮の妥当性を事後確認(0.515 < 0.584)

---

## 6. 結論と次の一手

- **「DCTがだめ」ではなかった**: だめだったのは「文脈化なしでDCT」。エンコーダを足せばスペクトル適応が起き、小K品質が5倍になり(H-A)、しかも学習レイテントという強い対抗馬に自由生成で勝つ(H-B棄却)。**「滑らかな信号を作る文脈化」+「順序付けと正則化を担う固定DCT基底」の分業**が現時点の最良構成
- 教師強制CEは自由生成品質の代理として信頼できない(B事例)。ボトルネック比較は必ず自由生成で行うこと
- 次の一手の候補: (1) 小K(1〜8)の平坦域を狙ったK分布の再設計や中間Kでのスケジュール学習、(2) 自由生成ギャップを埋めるscheduled sampling、(3) A構成でのK>64への拡張(累積エネルギー74%はまだ伸びる)
